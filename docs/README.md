# Docs Index

Use this folder for all project documentation and internal notes.

## Structure

- `BASE.md`: persistent rules and lessons learned.
- `pid-digitization-plan.md`: PID (as-built dossier) digitization vision, alignment with this repo, and phased engineering roadmap.
- `compliance-data-checklist.md`: yes/no inventory for compliance MVP (B-EPD fields, rules ownership, BIM↔site, optional SPARQL/timeline).
- `compliance-run-ttl.md`: append-only `data/<projectId>-compliance-events.ttl` format and `POST /api/compliance-run`.

## Writing Style

- Use Markdown.
- Keep sections small and easy to scan.
- Prefer headings and short bullet points.
